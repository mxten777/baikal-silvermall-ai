# 실버몰 관리자 매뉴얼

> 버전: 1.0.0 | 작성일: 2026-03-02  
> 대상: 실버몰 운영 담당자 (admin role 계정 보유자)

---

## 1. 관리자 계정 및 접속

### 1.1 로그인

1. 브라우저에서 `https://silvermall.co.kr/login` 접속
2. 관리자 이메일과 비밀번호 입력 후 로그인
3. 로그인 성공 시 `/admin` 대시보드로 자동 이동

> ⚠️ 일반 회원 계정으로는 `/admin` 접근 불가. 관리자 계정 발급은 DB `users.role = 'admin'` 설정 필요.

### 1.2 로컬 개발 환경 관리자 계정 생성

```sql
-- Supabase Studio(http://127.0.0.1:54323) → SQL Editor에서 실행
UPDATE public.users SET role = 'admin' WHERE email = '관리자이메일@example.com';
```

## 3. 상품 관리 (`/admin/products`)

### 3.1 상품 목록

- 검색창: 상품명, SKU, slug로 검색
- 필터: 카테고리 / 상태(active·draft·품절·판매중지) / 유형(구매·대여)
- 목록 컬럼: 썸네일 | 상품명 | 카테고리 | 가격 | 재고 | 상태 | 수정일

### 3.2 상품 등록

1. `새 상품 등록` 버튼 클릭
2. 기본 정보 입력:
   - **상품명** (필수, 검색에 노출)
   - **카테고리·브랜드** 선택
   - **판매 유형**: 구매 / 대여 / 구매+대여
   - **급여 품목 여부**: 체크 시 상품 카드에 `급여` 배지 표시
3. 가격 설정:
   - 정가 (`price`)
   - 판매가 (`sale_price`, 빈칸이면 정가로 판매)
4. 이미지 업로드:
   - 최대 10장 (첫 번째 이미지가 썸네일)
   - 권장 크기: 800×800px, JPG/PNG, 1MB 이하
5. 옵션 설정 (색상·사이즈 등, 선택사항)
6. 재고 수량 입력
7. 상태 선택: `draft`(준비중) → `active`(판매중)
8. **저장** 클릭

> 💡 **급여 품목 등록 시 유의사항**  
> `is_insurance = true`로 설정하면 상품 카드에 `급여` 배지가, 상품 상세에 본인부담금 예상금액이 자동 계산됩니다.  
> 급여 한도 기준: 구매 연간 160만원, 자기부담 15%

### 3.3 상품 수정

1. 목록에서 상품 클릭 → 수정 폼으로 이동
2. 필요 항목 수정 후 **저장**
3. 이미지 순서 변경: 드래그 앤 드롭

### 3.4 재고·가격 관리

- `/admin/products/[id]/inventory`에서 옵션별 재고 수량 및 가격 일괄 수정 가능
- 재고 0 설정 시 해당 옵션 "품절" 표시

### 3.5 상품 상태 변경

| 상태 | 설명 |
|------|------|
| `draft` | 준비중 – 고객에 노출 안 됨 |
| `active` | 판매중 – 정상 노출 |
| `out_of_stock` | 품절 – 노출되나 주문 불가 |
| `discontinued` | 판매중지 – 노출 안 됨 |

## 4. 주문 관리 (`/admin/orders`)

### 4.1 주문 목록 조회

- 검색: 주문번호, 회원 이름/이메일, 상품명
- 필터: 상태별 / 기간별 / 결제방법별

### 4.2 주문 상태 전환

| 현재 상태 | 다음 상태 | 조건 |
|-----------|-----------|------|
| `paid` | `preparing` | 발주 확인 후 수동 전환 |
| `preparing` | `shipped` | 송장번호 등록 시 자동 전환 |
| `shipped` | `delivered` | 배송 완료 확인 (수동 또는 택배사 연동) |
| `paid` / `preparing` | `cancelled` | 취소 요청 처리 |

### 4.3 송장 등록

1. 주문 상세(`/admin/orders/[id]`) 진입
2. **송장 등록** 버튼 클릭
3. 택배사 선택 + 운송장 번호 입력
4. **등록** → 자동으로 `shipped` 상태로 변경, 고객에게 알림 발송

### 4.4 주문 취소 처리

1. 주문 상세에서 **취소 처리** 버튼 클릭
2. 취소 사유 선택 (고객 요청 / 재고 부족 / 기타)
3. Toss Payments 환불 처리:
   - 전액 취소: 자동 환불 API 호출
   - 부분 취소: 환불 금액 직접 입력
4. 적립금·쿠폰 복원 여부 선택

### 4.5 교환·반품 처리

1. 고객이 마이페이지에서 교환/반품 신청
2. 어드민 주문 상세에서 신청 내용 확인
3. 수거 송장 입력 → 상태 `return_requested` → `returned`로 전환
4. 환불 또는 재발송 처리

## 5. 회원 관리 (`/admin/users`)

### 5.1 회원 검색

- 이름, 이메일, 전화번호로 검색
- 가입일, 주문 횟수, 적립금 잔액 확인

1. 회원 상세 → **적립금 지급** 버튼
2. 지급 금액과 사유 입력
3. **지급** → `points_ledger` INSERT + `users.point_balance` 업데이트

### 5.3 회원 등급 변경

- `users.role`: `user` ↔ `admin` (SQL 직접 변경 권장)

## 6. 쿠폰 관리

### 6.1 쿠폰 생성

| 항목 | 옵션 |
|------|------|
| 할인 유형 | 정액(amount) / 정률(percent) |
| 적용 범위 | 전체 상품 / 특정 카테고리 / 특정 상품 |
| 최소 주문금액 | 예: 30,000원 이상 |
| 유효기간 | 시작일 ~ 종료일 |
| 발급 수량 | 무제한 / 제한 수량 |

### 6.2 쿠폰 발급 방법

- **개인 발급**: 회원 상세 → 쿠폰 발급
- **일괄 발급**: 쿠폰 관리 → 대상 회원 CSV 업로드 (2차 기능)
- **코드 발급**: 공개 쿠폰 코드 생성 → 이벤트 페이지에 공지

1. 대시보드의 **미답변 문의** 클릭 또는 `/admin/inquiries`

| 컬럼 | 내용 |
|------|------|
| `admin_id` | 작업한 관리자 |

## 12. 긴급 대응

### 12.1 결제 오류 발생 시

1. Supabase Dashboard → `payments` 테이블에서 해당 `order_id` 조회
2. `raw` 컬럼(jsonb)에서 Toss 응답 코드 확인
3. Toss Payments 관리자 콘솔 (`https://dashboard.tosspayments.com`) 로그와 대조
4. 필요 시 수동 환불 처리

### 12.2 재고 오류 발생 시

```sql
-- 재고 현황 직접 확인
SELECT p.name, i.option_key, i.qty 
FROM inventory i 
JOIN products p ON p.id = i.product_id
WHERE p.slug = '상품-slug';

-- 재고 수동 수정
UPDATE inventory SET qty = [수량] WHERE product_id = '[product_id]';
```

### 12.3 비밀번호 재설정

- 로그인 페이지 → **비밀번호 찾기**
- 가입 이메일로 재설정 링크 발송 (Mailpit에서 확인: `http://127.0.0.1:54324`)
