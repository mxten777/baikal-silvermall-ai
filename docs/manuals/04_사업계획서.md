## 1. 사업 개요

### 1.1 사업 목적

실버몰은 **장기요양 수급자와 그 가족을 위한 복지용구 전문 온라인 쇼핑몰**입니다. 고령화 사회의 진전에 따라 급격히 증가하는 복지용구 수요에 대응하여, 국가 급여 지원 체계를 최대한 활용하면서 사용자가 최저 부담으로 필요한 용구를 구매·대여할 수 있는 원스톱 플랫폼을 제공합니다.

### 1.2 한눈에 보는 사업 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 실버몰 (SilverMall) |
| 사업 유형 | B2C 이커머스 (복지용구 전문) |
| 법인 설립 | 2026년 |
| 주요 고객 | 장기요양 1~5등급·인지지원등급 수급자 및 가족 |
| 수익 모델 | 직매입 판매·대여 마진 + 제조사·유통사 연계 |
| 초기 투자 | 약 3억 원 (제품 재고 + IT 구축 + 마케팅) |
| 3년 목표 매출 | 약 60억 원 |

## 2. 시장 분석

### 2.1 거시 환경 (PEST)

**정치(P)**
- 장기요양보험 급여 한도 지속 확대 (2024년 구매 연 160만원)
- 복지용구 급여 품목 확대 추세
- 재가 케어 중심 정책 강화 (시설→재가 전환)

**경제(E)**
- 고령층 소비 지출 증가
- 복지용구 시장 연평균 8~10% 성장 (2025~2030)
- 온라인 쇼핑 침투율 급상승 (65세 이상 모바일 구매 증가)

**사회(S)**
- 65세 이상 인구 2025년 기준 약 1,000만 명 돌파
- 장기요양 인정자 수 2024년 기준 약 110만 명, 연 7% 증가
- 돌봄 가족(보호자)의 디지털 접근성 증가

**기술(T)**
- LLM 기반 개인화 추천 적용 용이
- 국민건강보험공단 공개 API (급여 한도 실시간 조회) 활용 가능
- 모바일 우선 커머스 생태계 성숙

| 구분 | 2024년 | 2027년 (예상) |

|------|-------------|------|
| 종합 이커머스 | 네이버쇼핑, 쿠팡 | 복지용구 특화 부재, 급여 처리 불편 |
| 복지용구 전문몰 | 휠체어몰, 복지용구넷 | 구형 UI, 모바일 최적화 미흡 |
| 오프라인 대여점 | 동네 복지용구 센터 | 온라인 부재, 가격 불투명 |

### 2.4 목표 고객 세그먼트

| 재가 케어 담당 사회복지사 | 다수 어르신 용구 구매 결정권 | 기관 계정, 대량 할인 |
| 고령층 직접 구매 (65세+) | 대형 글씨·간편 UI 필요 | 큰 글씨 모드, 전화 상담 연계 |
| 의료·요양기관 | B2B 대량 발주 | 기업 계정, 세금계산서 |

## 3. 문제와 기회

### 3.1 현재 시장의 문제

1. **복잡한 급여 처리**: 급여 대상 품목인지, 본인부담금이 얼마인지 소비자가 스스로 계산해야 함
2. **분산된 정보**: 급여 한도, 등급별 품목, 가격 정보가 공단 홈페이지·오프라인 센터에 분산
3. **구형 UX**: 기존 전문몰들은 모바일 최적화가 미흡하여 가족·보호자 이용 불편
4. **개인화 부재**: 어르신의 건강 상태·등급에 맞는 맞춤 상품 추천 없음
5. **대여 접근성 부족**: 단기·월 렌탈 서비스가 오프라인 중심으로 운영

### 3.2 실버몰의 솔루션

| 문제 | 솔루션 |
|------|--------|
| 복잡한 급여 계산 | 본인부담금 자동 계산 + 연간 한도 관리 도구 |
| 분산 정보 | 급여 가이드 + FAQ 콘텐츠 통합 제공 |
| 구형 UX | 모바일 퍼스트, 큰 글씨 지원, 탭바 | 
| 개인화 부재 | 어르신 프로필 기반 AI 추천 엔진 |
| 대여 접근성 | 온라인 월 렌탈 신청·관리 통합 |

| 세면·목욕용품 | 목욕의자, 샤워카트 | 구매 |
| 배변처리용품 | 이동식 변기, 변기 보조기 | 구매 |
| 감각·인지기능 | 보청기, 점자 도구 | 구매 |

- **급여 한도 트래커**: 연간 사용 금액 추적 및 잔여 한도 알림

## 6. 기술 전략

### 6.1 기술 아키텍처 요약

- **프론트엔드**: Next.js 15 (App Router) + TypeScript + Tailwind CSS
- **백엔드**: Supabase (PostgreSQL + Auth + Edge Functions)
- **결제**: Toss Payments
- **AI 추천**: 프로필 기반 규칙 엔진 → LLM 고도화 (2차)

### 6.2 기술 경쟁력

| 영역 | 경쟁사 대비 우위 |
|------|----------------|
| 모바일 UX | 모바일 퍼스트 설계, TabBar 네비게이션 |
| 검색 | PostgreSQL FTS + 한국어 트라이그램 검색 |
| 성능 | ISR(Incremental Static Regeneration)로 빠른 페이지 로드 |
| 확장성 | 서버리스 아키텍처로 트래픽 급증 대응 |
| 개인화 | 어르신 프로파일 기반 추천 알고리즘 |

### 6.3 데이터 전략

1. **1차 (MVP)**: 구매 패턴·프로필 데이터 수집
2. **2차**: 추천 알고리즘 고도화 (협업 필터링)
3. **3차**: 공단 API 연동으로 실시간 급여 한도 조회
